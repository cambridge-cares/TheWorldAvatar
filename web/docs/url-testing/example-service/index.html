Example One

The image below uses the URL "/images/image-one.png", this should work when accessing the service directly, but will NOT work when accessed via the "/example" redirect as the root of the environment changes behind the redirect.

Example Two

The image below uses the URL "http://localhost:3000/images/image-two.png", this should work when accessing the service directly, and MAY work when accessed via the "/example" redirect. This is considered mixed content (loading a HTTP resources from within a HTTPS environment), however, many modern browser will automatically upgrade this to a HTTPS request (which may then fail if your external service does not support HTTPS). It is not recommended to rely on this.

Example Three

The image below uses the URL "./images/image-one.png", this should work when accessing the service directly, but SHOULD also work when accessed via the "/example" redirect as it's relative to the root of the environment.
